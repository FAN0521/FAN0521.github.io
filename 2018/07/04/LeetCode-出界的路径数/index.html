LeetCode:出界的路径数

发表于 2018-07-04 | 分类于 Algorithm |

摘要：给定一个 m × n 的网格和一个球。球的起始坐标为 (i,j) ，你可以将球移到相邻的单元格内，或者往上、下、左、右四个方向上移动使球穿过网格边界。但是，你最多可以移动 N 次。找出可以将球移出边界的路径数量。答案可能非常大，返回 结果 mod 109 + 7 的值。

Problem：

题目：给定一个 m × n 的网格和一个球。球的起始坐标为 (i,j) ，你可以将球移到相邻的单元格内，或者往上、下、左、右四个方向上移动使球穿过网格边界。但是，你最多可以移动 N 次。找出可以将球移出边界的路径数量。答案可能非常大，返回结果 mod 109 + 7 的值。

测试用例：

input output
m = 2, n = 2, N = 2, i = 0, j = 0 6
m = 1, n = 3, N = 3, i = 0, j = 1 12

说明:

  • 球一旦出界，就不能再被移动回网格内。
  • 网格的长度和高度在 [1,50] 的范围内。
  • N 在 [0,50] 的范围内。

解法一：动态规划

设置一个三维数组，dp[k][i][j],表示从（i，j）出界，并且最多走k步的路径数。到达该位置，可以从上下左右四个方向走过来，并且最多走k-1步。那么该位置就等于上下左右四个位置最多走k-1步的方案数的和。

时间复杂度：$O(n^3)$

空间复杂度：$O(n^3)$

解法二：动态规划

相当于枚举走1到N步分别有多少种方案可以出界，然后加和。dp[i][j]表示走k步时从（i，j）出界的方案数。每次建立一个临时数据temp[m][n]，遍历每一个位置，计算走k+1时从（i，j）出界的方案数，然后将temp赋值给dp，进行下一次遍历。如果遍历到边界的位置，就给res加上相应的方案数。最后返回res。
