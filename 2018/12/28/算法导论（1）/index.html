算法导论（1）

发表于 2018-12-28 | 分类于 读书笔记 | | 阅读次数:

摘要：算法导论第一章到第四章。主要内容有：渐近时间复杂度分析（$O, \Theta, \Omega$），递归算法及其时间分析（代入法，递归树法，Master法），红黑树的旋转插入删除。

第一章 绪论

算法是一系列解决问题的过程，能够对给定的输入在有限的时间内获得所要求的输出。

算法的五个特征：有穷性、确定性、输入、输出、可行性

算法分析包括时间复杂度和空间复杂度

传统分析方法：$\sum (指令执行时间 * 执行次数)$

\begin{align}
T(n) = & C_1n + C_2(n-1) + C_3(n-1) + C_4\sum_{j=2}^{n}t_j + \\
& C_5 \sum_{j=2}^n(t_j-1) + C_6 \sum_{j=2}^n(t_j-1) + C_7(n-1)
\end{align}

最好情况：输入数据为升序，A[i] <= key，所以$t_j=1$。$T(n) = An + B$

最坏情况：输入数据为降序，A[i] > key, 所以$t_j=j$。$T(n) = An^2 + Bn + C$

平均情况：$t_j = j/2$，$T(n) 和 n^2$相关

通常会关注最坏时间复杂度：

  • 算法运行时间的上限，也是算法改进的一个判断标准
  • 在很多问题中，由于缺少足够的信息，因此算法往往运行在最坏情况
  • 平均时间往往与最坏时间相同

增长速率：比如多项式时间的算法，增长速率由最高项决定。

算法正确性分析：循环不变式。先确定算法的循环不变式，然后检查此循环不变式在算法执行期间是否满足以下三个条件：

  • 初始状态：循环前循环不变式为真
  • 保持状态：如果本次循环前循环不变式为真，则本次循环后依然为真
  • 终止状态：退出循环时，循环不变式为真

第二章 渐近符号

$\Theta$符号：上下界

$O$符号：渐近上界，即可表示紧上界，也可表示松上界

$\Omega$符号：渐近下界

定理：对于任意的函数$f(n)$和$g(n)$，当且仅当$f(n)=O(g(n))$且$f(n)=\Omega(g(n))$时，$f(n) = \Theta(g(n))$

利用极限比较函数间的增长速度：

$$
\lim_{n \to \infty} \frac {f(n)}{g(n)}=\left{
\begin{align}
& 0 \ \ \ \ \Rightarrow f(n) < g(n) \ \ \ \ \Rightarrow f(n) = O(g(n)) \
& C \ \ \ \ \Rightarrow f(n) = g(n) \ \ \ \ \Rightarrow f(n) = \Theta(g(n)) \
& \infty \ \ \ \Rightarrow f(n) > g(n) \ \ \ \ \Rightarrow f(n) = \Omega(g(n))
\end{align}
\right.
$$

第三章 分治法及递归算法的分析方法

分治法的三个基本步骤：

  • 分解问题：把原问题分解为若干个与原问题性质类似的子问题
  • 求解子问题：不断分解子问题直到可方便的求出子问题的解为止
  • 合并子问题的解：合并子问题的解得到原问题的解

MERGE: $T(n) = An + B = \Theta(n)$

MERGE-SORT:

$$ T(n) = \left{
\begin{align}
& \Theta(1) & n <= C \
& aT(\frac{n}{b}) + D(n) + C(n) & n > C
\end{align}
\right.
$$

  • $\Theta(1)$表示常数时间
  • a为分解后的子问题的个数
  • 1/b为分解后子问题与原问题相比的规模
  • D(n)表示分解子问题所花费的时间
  • C(n)表示合并子问题解所花费的时间

对于MERGE-SORT：

$$a = 2, b = 2, D(n) = \Theta(1), C(n) = \Theta(n)$$

$$ T(n) = \left{
\begin{align}
& \Theta(1) & n = 1 \
& 2T(\frac{n}{2}) + \Theta(n) & n > 1
\end{align}
\right.
$$

等价于：

$$ T(n) = \left{
\begin{align}
& C & n = 1 \
& 2T(\frac{n}{2}) + Cn & n > 1
\end{align}
\right.
$$

递归树法：$T(n) = Cn\lg n + Cn = \Theta(n\lg n)$ (数学归纳法证明)

递推法：结果同上

分治法适用条件：

  • 原问题可以分解为若干个与原问题性质类似的子问题
  • 问题的规模缩小到一定程度后可方便求出解
  • 子问题的解可以合并得到原问题的解
  • 分解出的各个子问题应相互独立，既不包含重叠子问题

Master定理：对于非负递归函数$T(n) = aT(n/b) + f(n)$，其中a >= 1, b > 1且为常数，f(n)为渐近函数，则T(n)有如下三种渐近界限：

  • 如果 $f(n) = O(n^{\log_b{a-\epsilon}})$, 常数$\epsilon > 0$，则$T(n) = \Theta(n^{\log_b a})$
  • 如果 $f(n) = \Theta(n^{\log_ba})$, 则$T(n) = \Theta(n^{log_ba} \times \lg n)$
  • 如果 $f(n) = \Omega(n^{\log_b{a+\epsilon}})$, 常数$\epsilon > 0$且存在常数 c < 1, n足够大时有$af(\frac{n}{b}) <= cf(n)$, 则$T(n) = \Theta (f(n))$

PARTITION时间：$\Theta(n)$

QUICKSORT时间：$T(n) = T(q) + T(n-q-1) + \Theta(n)$

最坏情况：两个区间的大小分别为 n-1 和 0，$T(n) = \Theta(n^2)$

最好情况：$T(n) = O(n \lg n)$

平衡划分：假设每次划分按照9:1的比例划分，则时间为：$T(n) <= T(\frac{9n}{10}) + T(\frac{n}{10}) + \Theta(n)$, 构造递归树可得：$T(n) = cn \log _{\frac{10}{9}}n + cn = O(n \lg n)$

最坏情况：$T(n) = \max \limits _{0 \leq q \leq n-1} (T(q) + T(n-q-1)) + \Theta(n)$，猜测$T(n) = \Theta(n^2)$，替换法证明$T(n) \leq dn^2$

平均情况：定理：令X为排序n个元素时，partition算法所需要执行的总的比较次数，那么快速排序的执行时间为 $O(n+X)$。$X = \sum \limits _{i = 1} ^{n-1} \sum \limits {j=i+1} ^n X{ij}$，两边求期望可得$E[X] = O(n \lg n)$

快排中引入随机化是为了降低最坏情况的概率

第四章 红黑树

二叉查找树的性质：左子树小于等于根节点，右子树大于等于根节点

设高度为h，查找、插入、删除的时间为$O(h)$

红黑树的特性：

  • 它是一棵二叉查找树
  • 每个节点要么红色要么黑色
  • 根节点为黑色
  • 叶节点（nil）为黑色
  • 如果某个节点为红色，则它的左右孩子节点为黑色
  • 对树中任一节点，所有从该节点出发到其叶节点的路径中均包含相同数目的黑色节点

数据结构：p、key、left、right、color

黑高度：bh(x)表示从节点x出发(不包括x节点)到其叶节点路径上的黑色节点个数

定理：具有n个内部节点的红黑树高度最多为$2 \lg (n+1)$

红黑树的旋转：左旋和右旋

左旋：1.记录y；2.y的左子树连接到x的右子树；3.y连接到x.p；4.x连接到y的左子树

时间复杂度：O(1)

红黑树的插入操作：1.按照二叉查找树的方式插入新节点z；2.恢复红黑树的特性

  • case1: z的uncle节点y为红色
  • case2: z的uncle节点y为黑色，且z是右孩子
  • case3: z的uncle节点y为黑色，且z是左孩子

退出while循环后，根节点为红色的两种情况：

  • 新节点插入到空树中
  • 新节点的uncle节点为红色，且多次变换后uncle节点总为红色

红黑树高度$h = O(\lg n)$，RB-INSERT算法时间为$O(\lg n)$，RB-INSERT-FIXUP算法时间为$O(\lg n)$

红黑树的删除操作：1.按照二叉查找树的方式删除节点；2.恢复红黑树的特性。

  • case1: x的兄弟节点w是红色
  • case2: x的兄弟节点w是红色，且w的两个子节点都是黑色
  • case3: x的兄弟节点w是黑色，且w的左孩子是红色，右孩子是黑色
  • case4: x的兄弟节点w是黑色，且w的右孩子是红色

RB-DELETE时间复杂度：RB-DELETE-FIXUP 复杂度 $O(n \lg n)$. RB-DELETE 复杂度 $O(n \lg n)$
